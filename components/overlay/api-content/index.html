Attributes and Properties #

Property Attribute Type Default Description delayed delayed boolean An Overlay that is `delayed` will wait until a warm-up period of 1000ms has completed before opening. Once the warmup period has completed, all subsequent Overlays will open immediately. When no Overlays are opened, a cooldown period of 1000ms will begin. Once the cooldown has completed, the next Overlay to be opened will be subject to the warm-up period if provided that option. disabled disabled boolean Whether the overlay is currently functional or not offset offset number | [number, number] 0 The `offset` property accepts either a single number, to define the offset of the Overlay along the main axis from the trigger, or 2-tuple, to define the offset along the main axis and the cross axis. This option has no effect when there is no trigger element. open open boolean Whether the Overlay is projected onto the "top layer" or not. placement placement "top" | "top-start" | "top-end" | "right" | "right-start" | "right-end" | "bottom" | "bottom-start" | "bottom-end" | "left" | "left-start" | "left-end" Instruct the Overlay where to place itself in relationship to the trigger element. receivesFocus receives-focus 'true' | 'false' | 'auto' 'auto' Whether to pass focus to the overlay once opened, or to the appropriate value based on the "type" of the overlay when set to `"auto"`. tipPadding tip-padding number | undefined trigger trigger string | undefined An optional ID reference for the trigger element combined with the optional interaction (click | hover | longpress) by which the overlay shold open the overlay with an `@`: e.g. `trigger@click` opens the overlay when an element with the ID "trigger" is clicked. type type "auto" | "hint" | "manual" | "modal" | "page" 'auto' Configures the open/close heuristics of the Overlay.
